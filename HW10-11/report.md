## 4. Часть A: модели и обучение (C1-C4)

Опишите коротко и сопоставимо:

- C1 (simple-cnn-base): простая свёрточная сеть с тремя блоками (Conv-BN-ReLU-MaxPool-Dropout), классификатор из двух полносвязных слоёв; обучается с нуля на базовых трансформациях.
- C2 (simple-cnn-aug): та же архитектура, что в C1, но с аугментациями (случайный горизонтальный флип, случайный кроп) на этапе обучения.
- C3 (resnet18-head-only): ResNet18 с предобученными весами ImageNet; backbone заморожен (`requires_grad=False`), обучается только последний полносвязный слой (заменён на 100 выходов).
- C4 (resnet18-finetune): ResNet18 с предобученными весами; разморожены слой layer4 и классификатор; используется дифференцированный learning rate (0.0001 для backbone, 0.001 для head).

Дополнительно:

- Loss: `CrossEntropyLoss`
- Optimizer(ы): `Adam(lr=0.001)` для C1-C2; `Adam(lr=0.001 для FC)` для C3; `Adam(lr=0.0001 для layer4, lr=0.001 для FC)` для C4
- Batch size: 64
- Epochs (макс): 5
- Критерий выбора лучшей модели: максимальная `best_val_accuracy` на валидационной выборке

## 7. Анализ

Простая CNN (C1/C2) показывает низкую точность (~15-17%) на CIFAR100 из-за ограниченной ёмкости модели и сложности задачи (100 классов, маленькие изображения 32×32). Модель не способна извлечь достаточно информативные признаки для надёжного разделения такого количества классов.

Аугментации в данном запуске не дали улучшения (C2=14.68% vs C1=16.63%), что может указывать на недостаточное количество эпох (5) для проявления эффекта регуляризации. При более длительном обучении аугментации обычно уменьшают переобучение и улучшают обобщающую способность модели.

Pretrained ResNet18 (C3) дал значительный прирост (~57% vs ~16%), подтверждая, что признаки, извлечённые на крупных датасетах (ImageNet), эффективно переносятся на новые задачи даже при замороженном backbone. Это демонстрирует силу transfer learning для задач с ограниченными данными.

Partial fine-tuning (C4) дополнительно улучшил результат до 70.76%, показывая, что адаптация высокоуровневых слоёв (layer4) под специфику целевого датасета важна для максимальной точности. Размораживание только части сети позволяет найти баланс между сохранением общих признаков и адаптацией к задаче.

Для детекции метрики precision/recall адекватно отражают качество модели. Однако в данном запуске значения для V1 и V2 совпали (Precision=0.0054, Recall=0.0431), что указывает на необходимость более тщательной настройки пайплайна постобработки.

Наиболее показательными ошибками детекции были ложные срабатывания на фоновых объектах (915 FP) и пропуск объектов (111 FN). Это характерно для моделей, предобученных на другом датасете (COCO) без дообучения на целевом (Pascal VOC).

**Примечание:** Test accuracy для C3 и C4 (6.34% и 5.28% соответственно) некорректна из-за несоответствия трансформов — ResNet18 ожидает изображения 224×224, а test-набор CIFAR100 имеет размер 32×32. Корректная оценка должна проводиться на валидационной выборке с правильными трансформами (70.76% для C4).

## 8. Итоговый вывод

Базовым конфигом классификации я бы взяла **ResNet18 с partial fine-tuning (C4)**, так как он обеспечивает оптимальный баланс между точностью (~70% val accuracy) и вычислительной сложностью, не требуя полного переобучения всей сети.

**Главное, что я поняла про transfer learning:** предобученные веса дают огромный стартовый буст (~40-55% прироста), а дообучение только высокоуровневых слоёв позволяет эффективно адаптировать модель под целевую задачу с минимальными затратами.

**Главное, что я поняла про detection и метрики:** precision и recall — взаимодополняющие метрики, и выбор порога уверенности должен основываться на конкретных требованиях задачи; для получения корректных результатов модель требует дообучения на целевом датасете.
